# Папка «Отзывы» для сайта

Файлы `.md` или `.txt` здесь — это **тот же контент**, что и в папке **«Отзывы»** внутри портфолио на [Яндекс.Диске](https://disk.yandex.ru/d/ZDM9-AR8LwjYWw) и [Google Drive](https://drive.google.com/drive/folders/148LqiojSAFOb20QP93K4K0o3_jIbtCPK?usp=drive_link).

## Как обновить

1. Добавьте или измените текст на Диске / в Drive.
2. Скопируйте файл в эту папку `content/reviews/` (имя латиницей и цифрой в начале для порядка, например `04-novyy.md`).
3. Задеплойте сайт — на главной подтянется **число отзывов** и карусель (до3 случайных, если файлов больше трёх).

## Формат одного отзыва

- **Первая строка** — подпись (имя, компания, роль).
- **Пустая строка**.
- Дальше — текст отзыва.

Если пустой строки нет, весь файл считается текстом, подпись на сайте будет «Клиент».

Файлы `README*` и `_*` в список отзывов не попадают.

## Как это попадает на сайт

Перед `npm run build` и `npm run dev` автоматически выполняется **`npm run sync-reviews`**: из этой папки собирается **`data/reviews.json`**, и уже **он** попадает в сборку Next.js. На сервере во время работы сайта чтения папки `content/reviews` не нужно — достаточно задеплоить проект и дождаться `npm run build` на VPS.
