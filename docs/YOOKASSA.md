# Что нужно от ЮKassa для этого сайта

Оплата в корзине идёт **только через ЮKassa** (API v3). Ниже — что взять в личном кабинете и что прописать у себя.

## 1. Регистрация и магазин

1. Зарегистрируйтесь на [yookassa.ru](https://yookassa.ru/), пройдите проверку (как требует ЮKassa для вашего вида бизнеса).
2. Создайте **магазин** (приём платежей на сайт).
3. Дождитесь активации приёма платежей в **боевом** или используйте **тестовый** магазин/ключи для отладки (в кабинете есть тестовые данные).

## 2. Данные для `.env` на сервере (обязательно)

| Переменная | Где взять |
|------------|-----------|
| `YOOKASSA_SHOP_ID` | Личный кабинет → Настройки → **Идентификатор магазина** (число, например `123456`). |
| `YOOKASSA_SECRET_KEY` | Тот же раздел → **Секретный ключ** (строка; храните только на сервере, не в Git). |
| `NEXT_PUBLIC_SITE_URL` | Публичный URL сайта **без** слэша в конце, в проде TrueWeb: `https://www.truewebwork.ru`. Нужен для `return_url`: после оплаты пользователь вернётся на `https://…/payment/return`. |

Пример фрагмента `.env`:

```env
NEXT_PUBLIC_SITE_URL=https://ваш-домен.ru
YOOKASSA_SHOP_ID=ваш_shop_id
YOOKASSA_SECRET_KEY=ваш_секретный_ключ
```

После изменения `.env` перезапустите приложение (например `pm2 reload ecosystem.config.cjs --update-env`).

## 3. URL в личном кабинете ЮKassa

### HTTP-уведомления (вебхук), по желанию

Чтобы сервер получал события о платежах (часто нужно для учёта и автоматизации):

- **URL:** `https://ВАШ_ДОМЕН/api/yookassa/webhook`
- Включите в настройках магазина раздел про **HTTP-уведомления** / **Webhook** (название зависит от интерфейса).

Сейчас маршрут в проекте **принимает** уведомление и **пишет в лог**; при необходимости доработайте проверку по [документации ЮKassa](https://yookassa.ru/developers/using-api/webhooks) (IP, подпись).

### Возврат пользователя после оплаты

Отдельно в кабинете **не обязательно** указывать success/fail URL для нашей схемы: в API при создании платежа уже передаётся `return_url` = `{NEXT_PUBLIC_SITE_URL}/payment/return`.

## 4. Как сайт создаёт платёж

- Запрос: `POST https://api.yookassa.ru/v3/payments` (из серверного кода, см. `lib/yookassa-client.ts`).
- Тип подтверждения: `redirect` — пользователь уходит на страницу оплаты ЮKassa и после оплаты возвращается на `/payment/return`.
- Сумма **пересчитывается на сервере** по `lib/services-catalog.ts`, а не из браузера.

## 5. Тестирование без списания

- В `.env` можно использовать **тестовые** `shopId` и секретный ключ из кабинета (режим тестовых платежей).
- Либо включите **`PAYMENT_BYPASS=1`** / **`YOOKASSA_BYPASS=1`** только на стенде: тогда редирект пойдёт сразу на `/payment/return` без вызова API (в production так не делайте).
- Локально **`npm run dev`**: если **заданы** `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY`, создаётся настоящий платёж в ЮKassa (после оплаты вернёт на `NEXT_PUBLIC_SITE_URL/payment/return`). Если ключей нет — включается обход, как раньше.

## 6. Проверка перед продакшеном

- В браузере откройте `https://ВАШ_ДОМЕН/api/yookassa/webhook` — ожидается не GET-страница для проверки, вебхук принимает **POST**; для проверки используйте тестовый платёж из кабинета или логи сервера.
- Убедитесь, что с телефона/ПК корзина открывает страницу ЮKassa и после «успеха» возвращает на `/payment/return`.
- Реквизиты и документы на сайте (`/requisites`, `/privacy`, …) должны **совпадать** с данными в анкете ЮKassa.
